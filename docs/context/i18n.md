# i18n

## Locales
- `en` — English (default)
- `sk` — Slovak

## Library
**react-i18next** + `i18next` + `i18next-browser-languagedetector`.

## URL strategy
Locale lives in the URL as the first path segment: `/en/...`, `/sk/...`. The root `/` redirects to the detected or default locale. The `$lang` layout route validates the segment and sets the active language.

## Adding a string
1. Add the key to `src/i18n/locales/en.json`.
2. Add the translation to `src/i18n/locales/sk.json`.
3. Use `const { t } = useTranslation()` in the component.

## Adding a locale
1. Add the code to the supported list in `src/lib/locale.ts`.
2. Create `src/i18n/locales/<code>.json` with full key parity.
3. Add a language-switcher entry in `Header.tsx`.
4. Translate legal pages — `privacy-policy.tsx` and `terms-of-use.tsx` carry hardcoded content per locale, so each must be updated.
